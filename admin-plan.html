Contexte & Architecture
ℹ
Le panel admin utilise le même backend Express.js existant — pas de backend séparé. Des routes /api/admin/* seront ajoutées, protégées par un middleware dédié avec un JWT admin distinct. Le dossier /admin/ sera un nouveau workspace React/Vite à la racine du projet.
Backend (existant, étendu)

Express.js · TypeScript · Prisma · PostgreSQL · Redis
Ajout de routes /api/admin/* + modèle Admin + middleware isAdminAuthenticated

Admin Panel (nouveau dossier)

React 18 · Vite · TypeScript · TailwindCSS · Redux Toolkit + RTK Query · React Router v6 · Recharts

Module 1 — Authentification Admin

MéthodeRouteActionNotes
POST /api/admin/auth/login Connexion admin Rate limit 3 tentatives / 15 min · JWT signé ADMIN_JWT_SECRET

GET /api/admin/auth/me Profil admin courant Retourne firstName, lastName, isSuperAdmin
POST /api/admin/auth/logout Déconnexion Invalide le refresh token en DB

PHASE 1 — Backend : Fondations (Jours 1–3)

JOUR 1 ✓ (Fait)
Modèle Admin + Authentification backend
8h
  • Ajouter modèle Admin dans prisma/schema.prisma : id, email, password (Argon2), firstName, lastName, isSuperAdmin, refreshToken
    + champ isSuspended sur Organization
  • Créer et appliquer la migration Prisma : prisma migrate dev --name add_admin_model
  • Script seed admin : npm run seed:admin — crée le premier SuperAdmin via CLI
    Protection : échoue si un admin existe déjà
  • Variables d'environnement : ADMIN_JWT_SECRET, ADMIN_JWT_REFRESH_SECRET
  • Middleware admin.middleware.ts : isAdminAuthenticated + requireSuperAdmin
    JWT séparé du JWT user — impossibilité d'usurpation croisée
  • Controller + routes auth admin : login, refresh, me, logout
    Rate limit strict : 3 tentatives / 15 min
  • Tests : middleware auth · login valide/invalide · refresh token · rate limiting

Hypothèses & Risques

⚠
Risque principal : La révision de documents (preview PDF/images) peut nécessiter un ajustement des Content-Security-Policy headers Nginx pour autoriser les iframes ou les URLs S3. Prévoir +2h si les docs sont en S3 avec URLs signées.
ℹ
Hypothèse : Un seul développeur full-stack. Avec 2 développeurs (1 backend, 1 frontend), le délai peut descendre à 10-11 jours en travaillant en parallèle sur les phases 1-2 (backend) et 3-4 (frontend scaffolding).
✓
Avantage : Réutilisation de la stack complète (même TailwindCSS, même patterns RTK Query, même i18n) — pas de temps perdu en montée en compétence ou en choix technologiques.
